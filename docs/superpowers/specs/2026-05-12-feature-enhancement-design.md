# 功能增强 — 设计规格

## 概述

5 项增强：修复学生名显示 bug、批量排课、日历签到/请假/编辑、课程记录 Tab + 编辑、学生状态 + 筛选。

## 一、Bug 修复：Select 学生名显示

**问题：** @base-ui/react 的 `Select.Value` 选中后显示 ID（cuid）而非学生姓名。

**方案：** 在提交表单时用 `students.find()` 获取名称显示；或在 Select 外层包组件，手动渲染选中值。

## 二、批量排课

**数据模型：** 无需改表，复用 Schedule。

**UI：** 日历页新增「批量排课」按钮 → Dialog。

**表单：**
- 选择学生
- 开始日期 + 结束日期
- 多选星期（一~日，至少一个）
- 上课时间（默认 18:00）
- 时长（默认 45）

**计算预览：** `日期范围中匹配选中星期的天数 = N 节课`

**提交：** 批量创建 Schedule 记录。

## 三、日历课程列表操作

在选中日期的课程卡片上增加：

| 状态 | 显示 | 操作按钮 |
|------|------|----------|
| 待签到 | 橙色 "待签到" | 签到 / 请假 / 编辑 |
| 已签到 | 绿色 "已签到" | 编辑 |
| 已请假 | 灰色 "请假" | 编辑 |

**签到逻辑：** 创建 Lesson 记录（status=ATTENDED），该排课从日历移除。
**请假逻辑：** 创建 Lesson 记录（status=LEAVE），该排课从日历移除。
**编辑逻辑：** 打开 Dialog 修改排课时间/学生。

## 四、课程记录 Tab + 编辑

**数据来源更新：** 课程记录现在有两个来源：
1. 排课签到/请假自动生成（带 scheduleId）
2. 手动新增（status 默认 ATTENDED）

**页面改造：**
- Tab「未上课」：显示排课中尚未签到的课程（Schedule where date <= today）
- Tab「已上课」：显示已签到/请假/手动记录的课程（Lesson）
- 每条 Lesson 记录可编辑（Dialog 修改曲目/备注/作业/状态）
- 签到/请假操作也可在这里完成

**状态变更：** Lesson status 增加 SCHEDULED（仅用于展示逻辑，数据来自 Schedule 表）。

## 五、学生状态 + 筛选

**数据模型变更：** Student 表新增 `status` 字段：
```
status String @default("ACTIVE")
// ACTIVE | GRADUATED | DROPPED
```

**学生表单：** 新增「状态」选择（在读/毕业/肄业），默认在读。

**学生列表页：** 顶部筛选栏 — 全部 / 在读 / 毕业 / 肄业，默认在读。

**API 变更：** `GET /api/students?status=ACTIVE` 支持按状态筛选。

## 测试策略

- 单元测试：批量排课日期计算、状态筛选逻辑
- 组件测试：签到/请假按钮状态切换
- E2E：批量排课 → 日历签到 → 课程记录 Tab 查看
